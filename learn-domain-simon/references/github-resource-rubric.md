# Learning Resource Rubric

Use this reference to select and rank learning resources for a learning plan. Resources must follow the weekly learning sequence and attach to the relevant knowledge-map node.

## Ranking Criteria

Rank resources by learning value using these signals:

- Authority: official organization, known expert, respected community, or widely cited source.
- Maintenance: recent commits or releases, active issue handling, and compatibility with current tools.
- Documentation: clear README, structured tutorials, examples, diagrams, or curriculum.
- Practicality: runnable examples, projects, notebooks, datasets, templates, or exercises.
- Learning path: content progresses from basics to advanced material.
- Community validation: stars, forks, discussions, external references, or adoption.
- Scope fit: directly relevant to the learner's chosen field and level.

Popularity alone is not enough. A smaller official or well-maintained teaching resource can outrank a large but stale list.

## Resource Categories

Classify selected resources into these categories when possible:

- `路线图类`: visual roadmaps, learning paths, topic maps, or dependency maps.
- `课程类`: structured lessons, weekly curricula, quizzes, labs, or open course material.
- `项目实战类`: build-from-scratch tutorials, project lists, notebooks, demos, or case exercises.
- `awesome 清单类`: curated lists useful for discovery, not necessarily linear learning.
- `官方实现类`: official SDKs, frameworks, standards, examples, or reference implementations.
- `参考工具类`: tools, templates, datasets, checklists, or playgrounds that support practice.
- `视频类`: lectures, walkthroughs, conference talks, demos, or short tutorials.
- `书籍类`: books, book chapters, official guides, or legally accessible reading pages.

If a category has no credible resource, omit that category instead of forcing a weak link.

## Output Format

For each selected resource, include:

- Resource name and verified URL.
- Type and category.
- Week and map node it supports.
- Recommended learning stage: beginner, intermediate, advanced, or reference.
- Why it is valuable.
- How to study it: read order, exercises to run, or parts to skip.
- Suggested deliverable the learner can produce after studying it.
- Thumbnail or cover URL when available and appropriate for an interactive map.

## Search Strategy

Use multiple query shapes:

- `<field> github roadmap`
- `<field> github awesome`
- `<field> github course`
- `<field> github tutorial`
- `<field> github examples`
- `<field> github project based learning`
- `<field> github build your own`
- `<field> github official examples`
- `<field> github curriculum`
- `<field> official documentation`
- `<field> beginner video tutorial`
- `<field> recommended books`
- `<field> course syllabus`

For technical fields, also search for official implementations, standards bodies, and popular frameworks. For non-technical fields, prefer curated lists, playbooks, templates, case-study repositories, and open course materials.

## Weekly Matching

Do not dump all resources at the end. Match resources to the learning sequence:

- Week 1: prerequisites, setup, vocabulary, and orientation.
- Early weeks: official docs, beginner courses, and simple examples.
- Middle weeks: project tutorials, code repositories, and practice exercises.
- Later weeks: architecture, deployment, advanced references, and synthesis projects.

Each week should include only the resources that the learner can realistically use that week.

## Failure Mode

If network or GitHub search is unavailable:

- Say that live verification is unavailable.
- Do not fabricate links.
- Provide 5-8 concrete search queries the user can run.
- Explain how to evaluate the results using the ranking criteria above.
- Suggest which categories to search first based on the learner profile.
